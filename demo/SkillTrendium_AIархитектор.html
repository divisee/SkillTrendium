🔥 Неожиданные и растущие тренды


* Figma упоминается в 10.9% вакансий, что необычно для роли, ориентированной на разработку, а не дизайн интерфейсов.
* Adobe (10 mentions) и связанные инструменты (Photoshop - 4, After Effects - 5, Premiere - 5, Illustrator - 1) востребованы в 20.4% вакансий, указывая на потребность в визуализации результатов работы ИИ.
* Android и iOS (по 10 упоминаний) присутствуют в 9.1% вакансий, вероятно, для интеграции ИИ-решений в мобильные приложения.
* CapCut (4 упоминания) и Motion Design (3 упоминания) в 7.2% вакансий свидетельствуют о спросе на ИИ-архитекторов, работающих с видеоконтентом.
* SMM (2 упоминания) и TikTok (1 упоминание) в 2.7% вакансий указывают на растущую потребность в ИИ для маркетинга и социальных сетей.

📊 Ожидаемый стек (baseline)


* AI – 68 упоминаний (61.8%).
* Python – 10 упоминаний (9.1%).
* Windows – 10 упоминаний (9.1%).
* Nvidia RTX – 10 упоминаний (9.1%).
* Docker – 3 упоминания (2.7%).

🚀 Что набирает популярность


* LLM (2 упоминания) и RAG (2 упоминания) демонстрируют рост, хотя и с низкой базы, указывая на интерес к генеративным моделям.
* FastAPI (3 упоминания) набирает популярность как фреймворк для создания API для ИИ-приложений.
* Kubernetes (2 упоминания) и Helm (2 упоминания) востребованы для оркестрации контейнеров с ИИ-моделями.
* PostgreSQL (3 упоминания) используется для хранения и обработки данных, необходимых для обучения и работы ИИ.
* Redis (3 упоминания) применяется для кэширования и ускорения работы ИИ-приложений.

⚠️ Возможные blind spots


* TensorFlow и PyTorch отсутствуют, несмотря на их широкое использование в ИИ.
* Apache Kafka не упоминается, хотя часто используется для потоковой обработки данных.
* Spark не представлен, несмотря на его возможности для обработки больших данных.
* Erlang/Elixir не указаны, хотя подходят для создания отказоустойчивых ИИ-систем.
* Rust отсутствует, несмотря на его растущую популярность в системном программировании и ИИ.

💡 Рекомендации


* Освоить TensorFlow для работы с глубоким обучением.
* Изучить Apache Kafka для обработки потоковых данных в реальном времени.
* Рассмотреть Kubernetes для развертывания и управления ИИ-приложениями в production.
* Уделить внимание FastAPI для создания высокопроизводительных API для ИИ-моделей.
* Изучить RAG для реализации систем, использующих извлечение информации для улучшения ответов LLM.

* Отсутствуют TensorFlow и PyTorch в вакансиях, несмотря на их широкое применение в машинном обучении.
* Agentic AI активно продвигается Adobe, но не представлено в анализе вакансий.
